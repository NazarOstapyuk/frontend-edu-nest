# Login Page Design

## Overview

Сторінка авторизації для EduNest — системи управління школою. Окремий маршрут `/login`, що рендериться поза `AdminLayout` (без сайдбару і хедера).

## Layout

- Центрована біла картка на сірому фоні (`#f0f2f5`)
- Ширина картки: 400px, padding: 40px 36px
- Box-shadow, border-radius: 12px

## Вміст картки (зверху вниз)

1. **Логотип** — синє коло (`#1a237e`) з іконкою школи (MUI `SchoolIcon`)
2. **Назва** — "EduNest", `fontWeight: 700`, колір `#1a237e`
3. **Підзаголовок** — "Вхід до системи", сірий текст
4. **Поле "Логін"** — MUI `TextField`, `fullWidth`, валідація: обов'язкове
5. **Поле "Пароль"** — MUI `TextField`, `type="password"`, кнопка показати/сховати (`InputAdornment` + `IconButton`), валідація: обов'язкове, мінімум 6 символів
6. **Кнопка "Увійти"** — MUI `Button`, `variant="contained"`, `fullWidth`, колір `primary`

## Валідація

- Схема `yup`: обидва поля обов'язкові
- Підключена через `react-hook-form` + `@hookform/resolvers/yup`
- Помилки відображаються під полями через `helperText` / `error` props

## Маршрутизація

- Новий маршрут `/login` додається на рівні кореневого роутера поряд з `AdminLayout`
- Після успішного сабміту — `navigate('/')` (заглушка, бо auth API ще немає)
- `/` редиректить на `/login` поки немає авторизації (не входить у цей спек)

## FSD структура

```
src/pages/login/
  ui/LoginPage.tsx   — компонент сторінки
  index.ts           — публічний barrel export
```

## Поза скоупом

- Реальна авторизація через API (JWT, сесії) — окремий спек
- "Забули пароль?" — не потрібно
- "Запам'ятати мене" — не потрібно
- OAuth (Google/Microsoft) — не потрібно
